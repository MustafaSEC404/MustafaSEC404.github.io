CRITICAL

Hardcoded Authentication Bypass (Backdoor) via Source Code Disclosure

Business Risk: Complete Account Takeover (ATO) and full administrative access, leading to unauthorized data modification, exfiltration of sensitive organizational assets, and personal data leakage.

Technical Insight: A publicly accessible source code backup archive (srp.tar.gz) exposed the backend codebase. Source code review of the authentication logic (Auth_model.php) revealed a hardcoded static string backdoor constraint within an explicit logical OR (||) operator, bypassing standard password verification functions.

Impact: Remote unauthenticated attackers could arbitrarily authenticate into any active administrative or user account, completely bypassing password validation if the target username is known.

Mitigation: Immediately eliminate the static hardcoded password condition from the source code, rely strictly on secure password verification functions like password_verify(), remove public access to archives, and enforce strict server-level folder permissions.

CRITICAL

Stored Cross-Site Scripting (XSS) – Purchase Request Internal Notes

Business Risk: Potential session abuse, phishing attacks, and unauthorized access to sensitive procurement data.

Technical Insight: Stored XSS vulnerability exists in the “Internal Notes” field where user input is rendered without proper output encoding, allowing execution of arbitrary JavaScript in the browser of any user viewing the affected record.

Impact: Although session cookies are protected with HttpOnly, arbitrary script execution enables DOM data exfiltration, UI manipulation, and phishing attacks within the trusted application context.

Mitigation: Implement contextual output encoding, input sanitization, and Content Security Policy (CSP).
